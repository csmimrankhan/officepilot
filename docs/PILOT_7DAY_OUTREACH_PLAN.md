# 7-Day Pilot Outreach Plan

Daily tactical plan for launching the OfficePilot AI pilot program.

## Day 1 — Prep & Warm Outreach

**Goal**: Send first batch of warm outreach to people you already know

- [ ] Review the waitlist (check `/admin/waitlist` for existing signups)
- [ ] Send personal emails to 3-5 people you know: former colleagues, accountant friends, bookkeeper contacts
  - Use `PILOT_OUTREACH_MESSAGES.md` — email template
- [ ] Post on LinkedIn: "I'm building a local-first invoice tool and looking for pilot users"
  - Use channel-specific LinkedIn DM template
- [ ] Set up a demo environment (seed demo data, verify everything works)
- [ ] Test the full waitlist flow end-to-end

**Output**: 5 personal emails sent, 1 LinkedIn post

## Day 2 — Accountants & Bookkeepers

**Goal**: Target accounting firms and independent bookkeepers

- [ ] Find 10 accounting firms / bookkeepers on LinkedIn or Google Maps
- [ ] Send cold emails using the Accountant outreach template
- [ ] Post in 2 accounting/Bookkeeping Facebook groups or Reddit communities
  - Use Reddit/community template from outreach messages
- [ ] Follow up with any Day 1 replies

**Output**: 10 cold emails sent, 2 community posts

## Day 3 — Admin Teams & SME Owners

**Goal**: Target admin managers and small business owners

- [ ] Find 10 admin managers or SME owners (LinkedIn, local business directories)
- [ ] Send cold emails using the Admin Manager / SME Owner templates
- [ ] Post on WhatsApp to 3-5 contacts using the WhatsApp template
- [ ] Reply to any Reddit comments from Day 2

**Output**: 10 cold emails, 5 WhatsApp messages

## Day 4 — Qualification Calls

**Goal**: Convert interest into scheduled demo calls

- [ ] Review all replies and waitlist signups from Days 1-3
- [ ] Send follow-ups to anyone who expressed interest
- [ ] Schedule 3-5 qualification calls using the interview script
- [ ] Screen prospects using `PILOT_QUALIFICATION_QUESTIONS.md`
- [ ] Update admin waitlist status: mark as `contacted` or `demo_scheduled`

**Output**: 3-5 demo calls scheduled

## Day 5 — Demo Calls

**Goal**: Run demo calls and collect feedback

- [ ] Prepare using `DEMO_CHECKLIST.md`:
  - [ ] Seed demo data
  - [ ] Login as owner
  - [ ] Verify all features working
  - [ ] Test Excel export
  - [ ] Test audit log
  - [ ] Test version history
  - [ ] Test kill switch
  - [ ] Close other apps, mute notifications
- [ ] Run 3-5 demo calls using `PILOT_INTERVIEW_SCRIPT.md`
- [ ] Score each call using `PILOT_FEEDBACK_SCORECARD.md`
- [ ] Record objections and requested features
- [ ] Update admin waitlist status for each prospect
- [ ] Send thank-you emails within 24 hours

**Output**: 3-5 demos completed, scorecards filled

## Day 6 — Onboarding & Follow-up

**Goal**: Start onboarding interested pilot users

- [ ] Send download/access links to users who want to try
- [ ] Schedule 15-minute follow-up calls for each user (1 week after demo)
- [ ] Send personalized emails to users who are not yet ready ("check back in 3 months")
- [ ] Collect initial feedback from users who already tried
- [ ] Log all feedback in the Feedback Inbox
- [ ] Prioritize top 3 feature requests from feedback

**Output**: 2-3 users onboarded, 3 feature requests logged

## Day 7 — Review & Iterate

**Goal**: Assess first week, plan next steps

- [ ] Review waitlist stats: total signups, sources, conversion rate
- [ ] Review feedback: common themes, top requests, blockers
- [ ] Check bug reports: any critical issues?
- [ ] Check admin dashboard:
  - [ ] Waitlist summary: total entries, by status
  - [ ] Feedback inbox: all feedback logged
  - [ ] Bug reports: any unresolved
- [ ] Decide on pilot capacity: are we accepting more users or focusing on current?
- [ ] Plan Week 2:
  - If under capacity: more outreach (BPO/outsourcing firms)
  - If at capacity: focus on onboarding and retention
  - If blocked: fix critical issues before accepting more
- [ ] Update this document with lessons learned

**Output**: Week 1 summary report, Week 2 plan
